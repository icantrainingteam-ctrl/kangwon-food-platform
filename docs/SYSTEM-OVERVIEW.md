# 강원푸드 iCAN K-Food 플랫폼 — 시스템 전체 설명서

> 버전 2.0 | 2026-03-20
> "팔란티어식 온톨로지 기반 실시간 요식사업 통합 플랫폼"

---

## 1. 프로젝트 개요

### 1.1 배경
- **회사명:** 강원푸드 (iCAN 사업부)
- **식당명:** 강원 (KANGWON)
- **위치:** 필리핀
- **컨셉:** 한국인이 직접 서빙하는 K-Food 레스토랑
- **통화:** PHP (필리핀 페소)
- **언어:** 한국어 / English / Tagalog

### 1.2 비전
전통적인 한국 음식점을 **데이터 기반의 지능형 플랫폼**으로 전환합니다.
팔란티어(Palantir)의 Foundry처럼, 모든 비즈니스 객체(고객, 메뉴, 주문, 식자재, 직원)를
**온톨로지(지식 그래프)**로 연결하고, 실시간 데이터 파이프라인 위에서 의사결정을 자동화합니다.

### 1.3 핵심 차별화
| 기존 POS 시스템 | 강원 플랫폼 |
|----------------|-----------|
| 매출만 기록 | 모든 데이터를 온톨로지로 연결 |
| 사후 보고서 | 실시간 KPI + AI 예측 |
| 감 기반 메뉴 결정 | BCG 매트릭스 기반 메뉴 엔지니어링 |
| 수동 고객 관리 | RFM 자동 세그먼테이션 |
| 분절된 도구 | 단일 플랫폼에서 전략→전술→실행 |

## 3. 4개 앱 상세 설명

### 3.1 📊 매니저 대시보드 (apps/web)

**용도:** 점장/경영진이 매장 전체를 모니터링하고 의사결정하는 통합 관제 센터

**10개 뷰:**

#### Command Center (메인 대시보드)
- **실시간 KPI 4종:** 오늘 매출, 주문 건수, 평균 객단가, 고객 만족도
- **시간대별 매출 차트:** 10시~23시 매 시간 매출 막대그래프
- **테이블 맵:** 12개 테이블 실시간 상태 (빈 테이블/사용중/예약/정리중)
- **AI 인사이트:** 자동 생성된 경영 인사이트 (경고/기회/정보)
- **인기 메뉴 TOP 5:** 주간 기준 주문 수/매출 랭킹
- **어제 대비 변화율:** 매출 전일 대비 %

#### Menu Lab (메뉴 연구소)
- **BCG 매트릭스:** 모든 메뉴를 4분면으로 분류
  - ⭐ Star: 높은 인기 + 높은 마진 → 유지/강화
  - 🐄 Cash Cow: 높은 인기 + 낮은 마진 → 가격 인상 검토
  - ❓ Question Mark: 낮은 인기 + 높은 마진 → 마케팅 강화
  - 🐕 Dog: 낮은 인기 + 낮은 마진 → 제거/리뉴얼
- **시각화:** 인기도(X축) × 마진율(Y축) 산점도
- **상세 테이블:** 가격, 원가, 마진%, 주문수, 매출, 태그
- **기간 필터:** 7/14/30/90일 기간 선택

#### Customer Pulse (고객 분석)
- **RFM 세그먼테이션:** 고객을 자동으로 6개 세그먼트로 분류
  - 👑 VIP: 빈번한 방문 + 높은 소비
  - 💎 충성 고객: 꾸준한 방문
  - 😊 일반 고객: 보통 빈도
  - 🆕 신규 고객: 최근 첫 방문
  - ⚠️ 이탈 위험: 방문 감소 추세
  - 💤 이탈 고객: 60일 이상 미방문
- **평점 분포:** 1~5점 막대그래프
- **감성 타임라인:** 14일간 일별 평균 평점 변화
- **긴급 대응:** 2점 이하 미해결 리뷰 즉시 표시

#### Strategy Board (전략 관리)
- **OKR 트리:** Objective → Key Results → Tactics 계층 구조
  - 각 KR별 진척률 바 (% 표시)
  - 현재값/목표값 실시간 추적
- **캠페인 관리:** 마케팅 캠페인 ROI 추적
- **AI 어드바이저:** Claude가 데이터 기반 전략 인사이트 생성
  - 긴급/주의/기회/정보 4단계 분류
  - 구체적 액션 제안 포함

#### 주문 관리
- 오늘 전체 주문 카드형 표시
- 상태별 필터 (대기/접수/조리중/서빙대기/완료/결제/취소)
- 서비스 모드 표시 (카운터/태블릿/직원 주문)
- 실시간 SSE 갱신

#### 리포트
- 매출 트렌드 차트 (기간별)
- 결제수단별 매출 비율
- 카테고리별 매출 비율
- 직원별 서빙 성과 테이블

#### 주방 모니터
- KDS 앱(localhost:3002)을 iframe으로 임베드
- 매니저가 주방 상황을 원격 모니터링

#### 직원 관리
- **등급 시스템:** 루키(🥉) → 프로(🥈) → 마스터(🥇) → 소믈리에(⭐)
- 서빙 건수, 평균 평점, 관련 매출 표시
- 다음 등급까지 진척률 바
- **서빙 교육 체크리스트:** K-Food 서빙 7단계 (인사→안내→추천→서빙→확인→리필→인사)

#### 백오피스 (재무)
- **통합 전표 (Journal):** 모든 거래 내역 테이블
- **계정별 원장 (Ledger):** 자금원장별 Running Balance
- **일일 마감:** 오늘 수입/지출/순이익 + 마감 실행
- **마감 이력:** 과거 마감 기록
- **백업/복원:** JSON 파일 백업/다운로드
- **영수증 스캔:** Gemini AI로 영수증 OCR

#### 설정
- 3개국어 전환 (한국어/English/Tagalog)
- 결제 수단 관리 (추가/수정/삭제)
- 비용 카테고리 관리
- 테이블 관리 (12개 테이블 현황)
- 직원 관리 (추가/수정/PIN 설정)

### 3.2 📱 태블릿 주문 앱 (apps/tablet)

**용도:** 고객 테이블에 설치된 태블릿에서 직접 주문

**주문 흐름:**

```
환영 화면 → 메뉴 브라우징 → 메뉴 상세 → 장바구니 → 주문 확인
    → 주문 상태 추적 (접수→조리중→서빙준비→완료) → 피드백/리뷰
```

**화면별 기능:**

| 화면 | 기능 |
|------|------|
| **환영** | 강원 브랜딩, 메뉴 보기 버튼, 직원 호출, 3개국어 선택 |
| **메뉴 브라우징** | 카테고리 탭, 메뉴 카드 그리드, HOT/NEW 배지, 가격/조리시간 |
| **메뉴 상세** | 바텀시트 모달, 알레르기 정보, "이렇게 드세요" K-Food 가이드, 수량 선택 |
| **장바구니** | 아이템 목록, 수량 조절, 특별 요청 입력, 총 금액 |
| **주문 상태** | 4단계 진행 표시 (접수→조리→서빙준비→완료), 경과 시간, 추가 주문 버튼 |
| **피드백** | 별점(1~5), 한줄 리뷰, 제출 시 할인 쿠폰 즉시 발급 |

**특징:**
- PWA 지원 (홈 화면 추가 가능, 세로 모드)
- SSE로 주문 상태 실시간 업데이트
- 서빙 완료 시 "한국인 직원이 곧 서빙해드립니다!" 안내

### 3.3 🍳 주방 KDS (apps/kitchen)

**용도:** 주방에 설치된 대형 모니터에서 주문 관리

**기능:**
- **주문 카드 그리드:** 대기 중인 모든 주문을 카드로 표시
- **개별 메뉴 상태 관리:** 각 메뉴별 "조리 시작" → "완료" 버튼
- **경과 시간 색상:** 초록(10분 미만) → 노랑(10~15분) → 빨강(15분 초과)
- **우선순위:** 15분 초과 시 자동으로 "rush" 표시 + 빨간 테두리 깜빡임
- **서비스 모드 표시:**
  - 🔔 진동벨 #N → 픽업/서빙 (국대떡볶이식)
  - 🙋 테이블 #N → 직접 서빙 (한국인 직원)
- **통계:** 오늘 대기중/완료/전체 주문 수 실시간 표시
- **필터:** 전체/대기/조리중 탭

**디자인:**
- 다크 테마 (주방 가시성)
- PWA 지원 (가로 모드)
- 새 주문 시 알림 사운드

### 3.4 🔧 API 서버 (apps/api)

**용도:** 모든 프론트엔드의 데이터 허브

**11개 API 라우트 그룹:**

| 라우트 | 엔드포인트 | 기능 |
|--------|-----------|------|
| `/api/orders` | GET, POST, PATCH | 주문 CRUD, 상태 업데이트, 결제 |
| `/api/menu` | GET | 메뉴 목록 (카테고리별), 인기 메뉴, 가용성 토글 |
| `/api/tables` | GET, POST, PATCH | 테이블 상태, QR 세션 시작/종료 |
| `/api/kitchen` | GET, PATCH | 주방 대기열, 아이템 상태 업데이트, 통계 |
| `/api/finance` | GET, POST | 자금 원장, 거래 등록, 카테고리, 일일 마감 |
| `/api/customers` | GET, POST | 고객 목록, 체크인, RFM 세그먼트 통계 |
| `/api/strategy` | GET, POST, PATCH | OKR 트리, KPI 업데이트, 캠페인 |
| `/api/feedback` | GET, POST | 피드백 제출, 통계, 쿠폰 자동 발급 |
| `/api/staff` | GET, POST, PATCH | 직원 목록, PIN 로그인, 추가/수정 |
| `/api/analytics` | GET | KPI, 메뉴매트릭스, RFM, 감성, 매출트렌드, 직원성과 |
| `/api/insights` | GET, POST, PATCH | AI 인사이트 목록, 자동 생성, 읽음 표시 |

**SSE (Server-Sent Events):**
```
/ws?role=manager  → 매니저 대시보드 (모든 이벤트)
/ws?role=kitchen  → 주방 KDS (주문 이벤트만)
/ws?role=tablet   → 태블릿 (주문 상태 + 진동벨만)
/ws?role=serving  → 서빙 직원 (서빙 알림만)
```

## 4. 데이터베이스 설계 (28개 테이블)

### 4.1 테이블 그룹

```
📦 주문 시스템 (5개)
├── orders          주문 (주문번호, 테이블, 고객, 상태, 금액)
├── order_items     주문 항목 (메뉴, 수량, 가격, 조리 상태)
├── order_events    주문 이벤트 로그 (상태 변화 추적)
├── tables          물리적 테이블 (번호, 좌석, 상태, QR코드)
└── table_sessions  테이블 점유 세션 (세션 토큰, 인원수)

📦 메뉴 시스템 (4개)
├── menu_categories     메뉴 카테고리 (한식메인, 찌개, 구이, 퓨전, ...)
├── menu_items          메뉴 아이템 (이름, 가격, 원가, 조리시간, 태그)
├── menu_item_ingredients  레시피 (메뉴 ↔ 식자재 연결)
└── menu_recommendations   AI 추천 기록

📦 고객 시스템 (3개)
├── customers       고객 (전화번호, RFM 점수, 세그먼트, 방문 기록)
├── feedbacks       피드백 (평점, 코멘트, 감성분석 결과)
└── coupons         쿠폰 (코드, 할인율, 유효기간)

📦 재고 시스템 (4개)
├── ingredients             식자재 (이름, 재고량, 최소재고, 보관위치)
├── inventory_transactions  입출고 기록 (구매, 사용, 폐기)
├── suppliers               공급업체 (이름, 연락처, 평가)
└── purchase_orders         발주서 (공급업체, 항목, 금액)

📦 재무 시스템 (4개)
├── financial_sources   자금 원장 (현금, 법인카드, GCash 등)
├── expense_categories  비용 카테고리 (식자재-육류, 인건비 등 17개)
├── transactions        거래 내역 (수입/지출, 금액, 카테고리)
└── daily_closings      일일 마감 (수입/지출 합계, 잔액 스냅샷)

📦 직원 시스템 (1개)
└── staff  직원 (이름, 역할, PIN, 서빙실적, 평점)

📦 전략 시스템 (3개)
├── strategies      OKR (목표→핵심결과→전술→액션 계층)
├── campaigns       마케팅 캠페인 (예산, ROI, 타겟 세그먼트)
└── strategy_logs   전략 실행 로그

📦 온톨로지 시스템 (4개)
├── entities            엔티티 (고객, 메뉴, 식자재, 직원 등)
├── ontology_relations  관계 (ordered, contains, requires 등)
├── events              이벤트 스트림 (모든 비즈니스 이벤트)
└── insights            AI 인사이트 (카테고리, 심각도, 제안)
```

## 6. 서비스 모드 (3가지)

### 6.1 국대떡볶이식 (카운터)
```
카운터 주문 → 선결제 → 진동벨 수령 → 조리 → 진동벨 알림 → 한국인 직접 서빙
```
- **적용:** 점심 러시, 혼밥, 테이크아웃
- **장점:** 빠른 회전율

### 6.2 태블릿 주문식
```
QR 스캔 → 태블릿 메뉴 → 주문 → 직원 확인 → 조리 → 한국인 직접 서빙 → 추가 주문 → 피드백
```
- **적용:** 저녁 식사, 그룹, 데이트
- **장점:** 높은 객단가, 데이터 수집

### 6.3 직원 주문식
```
착석 → 직원이 메뉴 추천 → 주문 접수 → POS 입력 → 조리 → 직접 서빙
```
- **적용:** VIP, 룸, 한국음식 처음 접하는 고객
- **장점:** 최고 수준 고객 경험

## 7. 온톨로지 (팔란티어 접근)

### 7.1 개념

팔란티어의 핵심은 **모든 비즈니스 객체를 엔티티(Entity)로 정의하고,
엔티티 간의 관계(Relation)를 추적하여, 숨겨진 패턴을 발견**하는 것입니다.

### 7.2 엔티티 타입

| 타입 | 예시 | 속성 |
|------|------|------|
| customer | 010-1234-5678 고객 | RFM 점수, 선호 메뉴, 방문 패턴 |
| menu_item | 김치찌개 | 가격, 원가, 주문수, 평점 |
| ingredient | 김치 | 재고량, 원가, 공급업체 |
| supplier | 로컬마켓A | 평점, 카테고리, 연락처 |
| staff | 정민수 | 서빙 건수, 평점, 등급 |
| order | 주문 #47 | 금액, 테이블, 상태 |
| campaign | 주말 해피아워 | 예산, ROI, 타겟 |

### 7.3 관계 타입

| 관계 | 예시 |
|------|------|
| ordered | 고객 → 주문 |
| contains | 주문 → 메뉴 아이템 |
| requires | 메뉴 아이템 → 식자재 |
| supplied_by | 식자재 → 공급업체 |
| served_by | 직원 → 주문 |
| feedback_on | 피드백 → 주문 |
| triggers | 인사이트 → 전략 |

### 7.4 자동 온톨로지 구축 예시

고객이 "김치찌개 1, 소주 1"을 주문하면:

```
Customer(010-xxxx) ──[ordered]──▶ Order(#47)
                                    ├──[contains]──▶ MenuItem(김치찌개)
                                    │                  ├──[requires]──▶ Ingredient(김치 200g)
                                    │                  ├──[requires]──▶ Ingredient(돼지고기 150g)
                                    │                  └──[requires]──▶ Ingredient(두부 100g)
                                    ├──[contains]──▶ MenuItem(소주)
                                    └──[served_by]──▶ Staff(정민수)
```

이 관계들이 자동으로 `entities`와 `ontology_relations` 테이블에 저장됩니다.

## 8. AI 통합

### 8.1 Google Gemini (영수증 OCR)
- **모델:** Gemini 2.5 Flash
- **입력:** 영수증 사진 (base64)
- **출력:** 상호명, 날짜, 총액, 품목별 가격
- **사용처:** 백오피스 영수증 스캔

### 8.2 Claude API (감성 분석)
- **모델:** Claude Sonnet 4.6
- **입력:** 고객 리뷰 텍스트 + 평점
- **출력:** 감성(5단계), 점수(-1~1), 키워드, 개선 제안, 긴급도
- **사용처:** Customer Pulse

### 8.3 Claude API (전략 추천)
- **모델:** Claude Opus 4.6
- **입력:** 현재 KPI, 매출 트렌드, 메뉴 성과, 고객 피드백
- **출력:** 인사이트 제목, 분석, 추천 액션(영향도/노력도), 리스크
- **사용처:** Strategy Board AI 어드바이저

### 8.4 Claude API (메뉴 추천)
- **모델:** Claude Sonnet 4.6
- **입력:** 현재 메뉴 성과, 계절 식자재, 고객 선호
- **출력:** 추천 메뉴, 신메뉴 아이디어(한국-필리핀 퓨전), 매트릭스 인사이트
- **사용처:** Menu Lab
